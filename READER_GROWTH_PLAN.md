# Reader Growth Plan

## 1) Mission (proposed)

Engineering Journey helps developers become stronger data engineers through practical, real-world guides on pipelines, distributed systems, and cloud analytics.

### Content promise

- Explain complex topics simply.
- Publish implementation-focused posts with examples and trade-offs.
- Help readers ship better systems, not just passively read.

---

## 2) Current state audit (from existing pages)

## Home

Current strengths:

- Strong visual structure and responsive card feed.
- Search exists, categories/tags are available.

Gaps:

- No clear primary CTA above the fold (subscribe / start-here / featured series).
- No “new here?” onboarding path.

## About

Current strengths:

- Personal and clean design.

Gaps:

- Buttons are not clearly conversion-oriented (“Read Blog”, “About Me” are generic).
- Missing trust signals (experience, what readers will gain, posting cadence).

## Blog details

Current strengths:

- Great reading layout, metadata, tags/categories links.

Gaps:

- Missing end-of-article retention block (related posts, next read, newsletter CTA).
- Missing social proof and sharing prompts.

## Tags/Categories/Archive

Current strengths:

- Browsing architecture is present and connected.

Gaps:

- No post counts shown for tags/categories.
- No “featured” or “popular” grouping.

---

## 3) What to add next (priority order)

## Priority A (highest impact, low effort)

1. Add a clear hero CTA on Home:
   - Primary: “Subscribe for new posts”
   - Secondary: “Start here”
2. Add a “Start Here” section on Home:
   - 3 curated posts (beginner, intermediate, advanced).
3. Upgrade About page conversion blocks:
   - “Who this is for”, “What you’ll learn”, “How often I publish”.
4. Add end-of-post retention block:
   - Related posts by category/tag.
   - One-line newsletter CTA.

## Priority B (growth + engagement)

1. Add estimated value labels on cards:
   - “Hands-on”, “Architecture”, “Beginner-friendly”.
2. Add post counts for tags/categories.
3. Add a pinned/featured article module on Home.
4. Add canonical “series” pages (e.g., Data Pipelines 101).

## Priority C (authority + discoverability)

1. Add author profile section on blog detail pages.
2. Add richer SEO metadata:
   - FAQ schema for tutorial posts.
   - Better OG image consistency.
3. Add “Last updated” and changelog-style note for technical posts.

---

## 4) Suggested IA improvements

- Keep top nav simple:
  - Home, Start Here, Blogs, Categories, About
- Keep Tags as secondary (footer or inside blog pages), not primary nav.
- Add breadcrumbs on blog detail page:
  - Home > Category > Post

---

## 5) Conversion-focused copy examples

## Home hero

- Title: “Practical Data Engineering, One Real System at a Time”
- Subtitle: “Weekly tutorials and architecture notes you can apply immediately.”
- CTA 1: “Start Here”
- CTA 2: “Subscribe”

## About section

- “I write for engineers who are building real data systems in production.”
- “Expect guides with code, trade-offs, and deployment lessons.”

---

## 6) Metrics to track (simple KPI set)

1. CTR from Home to blog posts
2. Average engaged time on post pages
3. Return visitor rate (7/30 days)
4. Subscription conversion rate
5. % sessions that view 2+ posts

Use these metrics to validate each UX/content change.

---

## 7) 2-week execution plan

Week 1:

- Add Home CTA + Start Here block.
- Improve About copy and button intent.
- Add related posts section on blog detail.

Week 2:

- Add tag/category counts.
- Add featured post on Home.
- Add tracking events for CTA clicks and related-post clicks.

---

## 8) Final recommendation

You already have a solid technical foundation (Sanity + taxonomy + responsive layout). The fastest path to growth is not more pages first—it is:

1. stronger value proposition messaging,
2. better “next action” prompts on every key page,
3. retention loops after each article.

If you want, next step can be implementing Priority A directly in code.
